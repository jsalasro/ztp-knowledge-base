AAP_day2_OPS
=====================================================

This repository contains ansible code to automate the configuration of AAP using CasC(Configuration as Code) approach and automations for deploy Satellite-Capsule and mirror image repositories from an source organization to a destiny organization between different Quay instances.

# Requirements:

We built an EE image that contains all the necessary software to run the automations. The code to buildd this is image is availabe on the ./build_ee folder on this repository to future refference.

- pull the EE image from the internal image registry on the machine act as ansiblecontrole-node, in our case, the bastion host.
- Review and update the values of the the keys on the extra_vars file. the required vars are:

```yaml
vault_aap_username: "admin"
vault_aap_password: "CHANGEME" # value of this variable should match with the value defined on the VP values hub file
vault_aap_hostname: "DOMAIN_EXAMPLE_18.COM"
vault_aap_validate_certs: false
vault_manifest_file: "../files/aap_manifest.zip" # update the manifest fle when necessary, this manifest file is deployed during the casc configuration to activate the AAP license.
gitlab_token: "CHANGEME" # aap_day2_ops git repo PAT
vault_password: "CHANGEME" # password to decrypt variables used by the automation.
quay_robotaccount_ee: "ansible-automation-platform+aap_readonly"
quay_token_ee: "CHANGEME" # quay tobot account to access the EE custom image
satellite_capsule_admin_password: "CHANGEME" # password of admin user on the capsula vm, value of this variable should match with the value defined on the VP values hub file.
```

# Deploy CasC command
```bash
ansible-navigator run playbooks/casc_ctrl_config.yaml --ep --pp missing --pa='--authfile=$AUTH_FILE_PATH' --eei DOMAIN_EXAMPLE_15.COM/ansible-automation-platform/ee-supported-rhel9-custom:2.6 -m stdout --pae false --lf /dev/null -- -vv -J
```

Ansible Automation - Mirroring Quay repositories
=====================================================

All the required objects to run this automation on AAP will be automatically create by the CaC playbook. A job template called _quay_site_mirror_ is create during the CasC execution and should be used to mirror the repositories from an organization between two quay registries. The following variables will be requested when launch the job template from the AAP UI.

```yaml
    source_quay_hostname: "example-registry-quay-openshift-operators.apps-crc.testing"
    source_quay_token: "EXAMPLE_TOKEN"
    source_quay_organization: "ocp"
    source_quay_username: "admin"
    source_quay_password: "EXAMPLE_PASSWD"

    quay_hostname: "example-registry-quay-openshift-operators.apps-crc.testing"
    quay_org_app_token: "CHANGEME"
    organization: "ocp1"
    organization_mail: "admin-ocp1@localdomain"
    robot_account: "ocp1+robotocp1"
```

Ansible Automation - Satellite Capsule installation
=====================================================

The CasC execution automatically creates a job template named satellite-capsule install, which is used for configuring and installing the Satellite Capsule software.

To successfully integrate the capsule with the existing Satellite infrastructure, specific information and files are needed. The customer must run the appropriate Satellite command to generate the activation key, secret key, and a necessary tar file. 

These outputs are then utilized by the capsule installation playbook for the correct configuration and installation of the capsule server.

NOTE: Generating these required files falls outside the scope of this automation and is the customer's responsibility, as we lack access to the Satellite application.

You must create or replace the vault_satellite_capsule_install_preprod1.yaml file at the root of the repository. This file is imported by the playbook and must contain the correct values for the required automation variables. 

Ensure you upload the .tar file to the ./files folder on the root of this repository before running` the job template.

```yaml
capsule_fqdn: "DOMAIN_EXAMPLE_19.COM"
satellite_fqdn: "DOMAIN_EXAMPLE_20.COM"
custom_ca_file: "ca.crt"
capsule_tar_file: "DOMAIN_EXAMPLE_19.COM-certs.tar" # File generated by the satellite server
capsule_tar_file_path: "/home/admin" # The path on the managed node where the file will be copied to
consumer_key: "CHANGEME" # Provided by the customer
consumer_secret: "CHANGEME" # Provided by the customer

# Provided by the customer
registration_command: |
  CHANGEME

capsule_install_options:
  - '--certs-tar-file'             "{{ capsule_tar_file_path + "/" + capsule_tar_file }}"'
  - '--foreman-proxy-register-in-foreman' "true"'
  - '--foreman-proxy-foreman-base-url'    "https://{{ satellite_fqdn }}"'
  - '--foreman-proxy-trusted-hosts'       "{{ satellite_fqdn }}"'
  - '--foreman-proxy-trusted-hosts'       "{{ capsule_fqdn }}"'
  - '--foreman-proxy-oauth-consumer-key'  "{{ consumer_key }}"'
  - '--foreman-proxy-oauth-consumer-secret' "{{ consumer_secret }}"'
```
